ESP32 AI 자율주행 RC카

실습 필요 사항

  1. 하드웨어 펌웨어 업로드
    아래 링크의 가이드를 참고하여 ESP32에는 [시리얼 제어], ESP32-CAM에는 [텔넷 스트리밍] 코드를 각각 업로드하세요.
    🔗 원격 조종 자동차 가이드 링크
  2. Anaconda 환경 설정
    Anaconda Prompt를 실행하고 아래 명령어를 이용하여 가상환경 설정과 필요한 라이브러리를 설치합니다.
    # 1. 가상환경 생성 및 활성화
    conda create -n drive_env python=3.8.20 -y
    conda activate drive_env

    # 2. 주피터 노트북 커널 설치
    pip install ipykernel
    python -m ipykernel install --user --name=drive_env

    # 3. 필요 라이브러리 설치 (Tensorflow 버전 필수 준수)
    pip install tensorflow==2.10.0 PyQt5 numpy pandas tqdm Pillow opencv-python matplotlib scikit-learn

    ※ drive_env는 가상환경의 이름입니다. 원하시는 다른 이름으로 변경하여 사용할 수 있으며, 이름을 바꿀 경우 위 명령어에 포함된 모든 drive_env를 동일하게 수정해 주세요.

위 준비가 끝났다면 아래 순서대로 주피터 노트북 파일을 다운로드하여 실습을 진행합니다.

  1. 텔넷 키보드 주행 (기본 제어 확인) 카메라 영상이 나오는지 확인하고, 키보드로 RC카를 직접 조종하여 회로 연결 상태를 점검합니다.
  2. 주행 데이터 수집 직접 운전하며 학습용 데이터를 수집합니다. 새로운 주행 데이터 생성을 원하면, 'N'키를 눌러 새 폴더를 만든 후 주행하세요.
  3. 티처블 머신 모델 테스트 (PC용) 2번 단계에서 수집한 데이터를 이용하여 구글 Teachable Machine에서 만든 keras_model.h5 파일을 PC에 저장된 이미지로 테스트해봅니다.
  4. 티처블 머신 자율주행 티처블 머신 모델을 RC카에 적용하여 바로 자율주행을 시도합니다.
  5. CNN 모델 학습시키기 2번 단계에서 수집한 데이터를 불러와 CNN 모델을 학습시키고 model_CNN.h5를 만듭니다.
  6. CNN 모델 시각화 학습된 CNN 모델을 시각화(Heatmap)하여 원리를 확인합니다.
  7. CNN 모델 자율주행 5번 단계에서 학습시킨 CNN 모델(model_CNN.h5)을 로드하여 RC카의 자율주행을 시도합니다.
  8. CNN 모델 멀티스레드 자율주행 학습시킨 CNN 모델을 로드하고 영상 수신과 AI 판단을 동시에 처리(멀티스레드)하여 반응 속도를 높인 코드입니다.
  9. EfficientNetB0 전이 학습 및 주행 2번 단계에서 수집한 데이터와 EfficientNetB0 모델을 이용하여 정확도를 높이는 심화 과정입니다. 아래 코드를 순서대로 활용하세요.
    • ① EfficientNetB0 전이 학습 코드 (weights_EN.h5 생성)
    • ② EfficientNetB0 자율주행 코드
    • ③ EfficientNetB0 멀티스레드 자율주행 코드
